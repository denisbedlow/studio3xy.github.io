title: Claude Context Cheat Sheet
description: The mechanisms Claude uses to receive context — what they are and how they relate.
---

## The five named mechanisms

<div class="card-grid">
  <div class="card">
    <strong>Skills</strong>
    <p>Markdown instruction files that teach Claude <em>how</em> to do a specific task well. Loaded automatically when a relevant task is triggered.</p>
    <div class="chips">
      <span class="chip">Persistent</span>
      <span class="chip">Text only</span>
      <span class="chip">No network</span>
      <span class="chip">Auto-loaded</span>
    </div>
  </div>
  <div class="card">
    <strong>Connectors</strong>
    <p>OAuth-based MCP integrations that give Claude access to external services (Gmail, Drive, Supabase…). Each connector = one service.</p>
    <div class="chips">
      <span class="chip">MCP-powered</span>
      <span class="chip">OAuth</span>
      <span class="chip">Read & write</span>
      <span class="chip">Live data</span>
    </div>
  </div>
  <div class="card">
    <strong>Plugins</strong>
    <p>Installable packages that bundle Skills + Connectors + slash commands + sub-agents into one unit. Distribution format for teams. Launched Jan 2026.</p>
    <div class="chips">
      <span class="chip">Bundles everything</span>
      <span class="chip">Team-oriented</span>
      <span class="chip">Marketplace</span>
    </div>
  </div>
  <div class="card">
    <strong>Memories</strong>
    <p>Short factual summaries auto-generated by Claude from past conversations. Tell Claude <em>who you are</em> across sessions. Editable by the user.</p>
    <div class="chips">
      <span class="chip">Auto-generated</span>
      <span class="chip">Cross-session</span>
      <span class="chip">Personal context</span>
    </div>
  </div>
  <div class="card">
    <strong>MD files</strong>
    <p>Any markdown document handed to Claude. Raw reference material. No special structure required — just content.</p>
    <div class="chips">
      <span class="chip">Per-session</span>
      <span class="chip">Manual</span>
      <span class="chip">General purpose</span>
    </div>
  </div>
</div>

## Other context sources

<div class="card-grid">
  <div class="card">
    <strong>Projects</strong>
    <p>Persistent instruction field scoped to a Project. Like a mini system prompt you write intentionally for a specific context.</p>
    <div class="chips">
      <span class="chip">claude.ai</span>
      <span class="chip">Scoped</span>
      <span class="chip">Persistent</span>
    </div>
  </div>
  <div class="card">
    <strong>User preferences</strong>
    <p>Global tone, formatting, and behavior settings defined in Settings. Apply across all conversations.</p>
    <div class="chips">
      <span class="chip">Global</span>
      <span class="chip">Persistent</span>
      <span class="chip">User-written</span>
    </div>
  </div>
  <div class="card">
    <strong>Uploaded files</strong>
    <p>PDFs, images, spreadsheets, code files. Attached to a session and read by Claude. Ephemeral.</p>
    <div class="chips">
      <span class="chip">Per-session</span>
      <span class="chip">Any format</span>
    </div>
  </div>
  <div class="card">
    <strong>Pasted content</strong>
    <p>Text, code, or data dumped directly into the chat. Simplest possible way to add context.</p>
    <div class="chips">
      <span class="chip">Per-session</span>
      <span class="chip">Manual</span>
      <span class="chip">Instant</span>
    </div>
  </div>
  <div class="card">
    <strong>Artifacts</strong>
    <p>Content created within a session remains referenceable and can be iterated on during that conversation.</p>
    <div class="chips">
      <span class="chip">Per-session</span>
      <span class="chip">Generated</span>
    </div>
  </div>
  <div class="card">
    <strong>Search / fetch results</strong>
    <p>When Claude searches the web or fetches a URL, that content enters the context window as live data.</p>
    <div class="chips">
      <span class="chip">Per-session</span>
      <span class="chip">Live</span>
      <span class="chip">External</span>
    </div>
  </div>
  <div class="card">
    <strong>System prompt</strong>
    <p>The foundational API mechanism. Operators inject instructions before the conversation starts. Everything else is an abstraction on top of this.</p>
    <div class="chips">
      <span class="chip">API only</span>
      <span class="chip">Operator-level</span>
      <span class="chip">Foundational</span>
    </div>
  </div>
  <div class="card">
    <strong>Tool results</strong>
    <p>Outputs from MCP tools, code execution, or any other tool call. All land in the context window as content Claude can reason over.</p>
    <div class="chips">
      <span class="chip">API / MCP</span>
      <span class="chip">Per-session</span>
      <span class="chip">Dynamic</span>
    </div>
  </div>
</div>

> All of these — without exception — are text injected into the context window. The context window is the one real primitive. Every mechanism above is just a convention for *how* and *when* to put text into it.
